# 启发式搜索
启发式搜索是盲目搜索的改进，利用启发式函数（heuristic function）来评估每个节点的优先级，从而指导搜索过程更快地找到目标节点。启发式函数通常是一个估计值，表示从当前节点到目标节点的距离或成本。

## 启发式函数
启发式函数通常用符号 `h(n)` 表示，其中 `n` 是当前节点。启发式函数的设计对于搜索算法的效率至关重要。一个好的启发式函数应该满足以下条件：
1. **可计算性**：启发式函数应该能够在合理的时间内计算出来。
2. **一致性**：启发式函数应该满足一致性条件，即对于任何节点 `n` 和其子节点 `n'`，启发式函数应该满足 `h(n) <= c(n, n') + h(n')`，其中 `c(n, n')` 是从节点 `n` 到节点 `n'` 的实际成本。
3. **可接受性**：启发式函数应该满足可接受性条件，即对于任何节点 `n`，启发式函数应该满足 `h(n) <= h*(n)`，其中 `h*(n)` 是从节点 `n` 到目标节点的实际成本。这决定了算法是否能保证找到最优解。

## 启发式搜索算法
启发式搜索算法包括A\*算法、贪心最佳优先搜索和IDA\*算法等。
### A\*算法
A\*算法是BFS的改进，它结合了实际成本 `g(n)` 和启发式函数 `h(n)` 来评估每个节点的优先级。A\*算法通过维护一个优先队列来选择下一个要扩展的节点，优先级由 `f(n) = g(n) + h(n)` 决定。A\*算法能够保证找到最优解，前提是启发式函数满足可接受性条件。
算法步骤如下：
1. 初始化：将起始节点加入优先队列，并设置 `g(start) = 0` 和 `f(start) = h(start)`。
2. 循环：从优先队列中取出优先级最低的节点 `n`。
   - 如果 `n` 是目标节点，则返回路径。
   - 否则，扩展节点 `n` 的所有子节点 `n'`。
     - 计算 `g(n') = g(n) + c(n, n')` 和 `f(n') = g(n') + h(n')`。
     - 如果 `n'` 不在优先队列中，或者 `g(n')` 更小，则将 `n'` 加入优先队列，并更新其优先级。
     

### 贪心最佳优先搜索
贪心最佳优先搜索只考虑启发式函数 `h(n)` 来评估节点的优先级，而不考虑实际成本 `g(n)`。这种算法可能会更快地找到一个解，但不保证找到最优解。
### IDA\*算法
IDA\*算法是DFS的改进，它结合了A\*算法的启发式评估和DFS的深度优先搜索。IDA\*算法通过设置一个阈值来限制搜索的深度，并在每次迭代中增加阈值，直到找到目标节点或达到最大迭代次数。
